# Performance budget lab

Chromium E2E + unit gates for Dual Read’s page-impact budgets.

## What is measured

| Budget | How | Default CI | `PERF_STRICT=1` |
|--------|-----|------------|-----------------|
| 5k initial index | `ContentSession` `perf.lastIndexMs` (active CPU, excludes yield waits) (+ vitest smoke) | ≤300ms E2E | ≤100ms (+5ms slack) |
| Mutation index | SPA nav → `perf.lastMutationIndexMs` | ≤80ms | ≤16ms |
| CLS (article + dense) | `PerformanceObserver` `layout-shift` | ≤0.35 | ≤0.05 |
| Mock first visible paint | translate start → first `.dual-read-target` | ≤900ms | ≤500ms |
| 20k long task | `longtask` entries (`PERF_FULL`) | ≤200ms | ≤50ms |
| 30× SPA heap | `performance.memory` (`PERF_FULL`) | ≤25% growth | ≤10% |

### CLS mitigations

Bilingual companions stay **in-flow** (product choice). Cumulative CLS is reduced by:

1. **Stable shell** — block mounts set `min-height` from host height (`data-dual-read-shell`) before text fill.
2. **Tighter block margins** in `dual-read.css`.
3. **Initial render coalesce** — longer buffer during first viewport drain (`RENDER_BUFFER_INITIAL_MS`).

Dense fixture: `e2e/fixtures/pages/lab-cls-dense.html` (~40 paragraphs).

### Index long-task slicing

The initial full-document index is split across cooperative yields on large pages:

1. **`collectUnitsAsync`** — same passes as `collectUnits`, yields via `scheduler.yield` / MessageChannel whenever a soft CPU budget (~12ms) is exceeded.
2. **`ContentSession.indexScopes`** — registers / observes units under the same CPU budget.
3. **`perf.lastIndexMs`** — sums active collector+register CPU only (yield waits excluded) so 5k budgets stay meaningful.

### Collector hotspot cache

Within one `collectUnits` / `collectUnitsAsync` pass:

1. **Memoize** `getComputedStyle` self-hide, `isVisible`, and layout-box checks (`WeakMap`).
2. **Invalidate** the memo across cooperative yields (avoid stale answers after mutations).
3. **Leaf fast path** — blocks with only text nodes skip `TreeWalker`.
4. **`hasChildBlock`** — cheap rejects before `extractText`.

On Chromium lab fixtures: 5k ~61ms, 20k ~239ms active index CPU.


## Commands

```bash
# Unit (jsdom collector smoke) — part of npm test
cd extension && npm test -- tests/collector-perf.test.ts

# E2E lab (requires chrome-mv3 build)
cd extension && npm run build && npm run e2e:perf

# Plan-strict ceilings (includes dense CLS ≤ 0.05)
PERF_STRICT=1 npm run e2e:perf

# + 20k long-task + 30× SPA heap
PERF_FULL=1 npm run e2e:perf:full

# From repo root
make test-e2e-perf
```

Scheduled CI: [`.github/workflows/nightly.yml`](../.github/workflows/nightly.yml) runs `PERF_STRICT` + `PERF_FULL` (and a release dry-run) daily / on `workflow_dispatch`.

Synthetic fixtures `lab-perf-5k.html` / `lab-perf-20k.html` are generated by the mock server (not checked into git).
